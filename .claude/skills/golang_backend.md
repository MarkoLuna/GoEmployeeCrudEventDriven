# Skill: Go Backend Development

> This skill applies to every Go module in **GoEmployeeCrudEventDriven**
> (`common/`, `employee-service/`, `employee-consumer/`).
> All code examples are drawn from or consistent with patterns already present in the repo.

---

## 1. Project Layout

Follow the [Standard Go Project Layout](https://github.com/golang-standards/project-layout).

```
<service>/
├── cmd/api-server/main.go     # Entry point only — no logic here
├── internal/
│   ├── app/                   # Application struct, DI wiring, server start
│   ├── config/                # One file per concern (db, kafka, auth, cors, logging)
│   ├── controllers/           # HTTP handlers — bind, validate, call service, respond
│   ├── services/              # Interfaces + concrete service structs
│   │   ├── impl/              # Implementations of service interfaces
│   │   └── stubs/             # Test stubs implementing the interface
│   ├── repositories/          # DB interfaces + implementations + stubs
│   ├── models/                # DB entity structs (no business logic)
│   └── routes/                # Route registration helpers
├── docs/                      # Auto-generated by swaggo/swag — never edit by hand
├── resources/
│   ├── init.sql               # DB schema bootstrap
│   └── ssl/                   # TLS certs (gitignored in production)
└── Makefile
```

### Rules
- `cmd/` contains **only** `main.go` — it creates the app struct and calls `app.Run()`.
- Nothing inside `internal/` is ever imported by another service module.
- Cross-service shared types live **exclusively** in `common/` and are imported via the `replace` directive.
- `pkg/` is reserved for packages that would be safe to publish externally; leave empty until needed.

## 2. Dependency Injection & the Application Struct

Wire all dependencies manually in `internal/app/application.go`. There is no DI framework.

```go
// internal/app/application.go
type Application struct {
    EchoInstance       *echo.Echo
    DbConnection       *sql.DB
    EmployeeService    services.EmployeeService        // interface
    EmployeeRepository repositories.EmployeeRepository // interface
    EmployeeController controllers.EmployeeController
    KafkaProducerService services.KafkaProducerService // interface
}
```

**Pattern**:
1. `main.go` constructs an `Application{}` with zero values.
2. `main.go` calls a bootstrap function (e.g., `app.Init()`) that creates and assigns all dependencies in dependency order.
3. `app.Run()` registers routes, starts the server.

```go
// cmd/api-server/main.go — keep this short
func main() {
    godotenv.Load()
    app := app.Application{EchoInstance: echo.New()}
    app.Init()
    app.Run()
}
```

### Constructor conventions

Use `New<Type>` factory functions that accept **interfaces**, not concrete types:

```go
// ✅ Accepts an interface — easy to test with a stub
func NewEmployeeController(svc services.EmployeeService) EmployeeController {
    return EmployeeController{employeeService: svc}
}

// ❌ Accepts a concrete type — couples caller to implementation
func NewEmployeeController(svc impl.EmployeeServiceImpl) EmployeeController { ... }
```

Factory functions for repositories **return the interface**, not the concrete type:

```go
func NewEmployeeRepository(db *sql.DB, initDb bool) EmployeeRepository { // ← interface return
    repo := &EmployeeRepositoryImpl{db}
    if initDb {
        repo.CreateTable()
    }
    return repo
}
```

## 3. Interface Design

### Declare interfaces close to the consumer, not the implementor

```
internal/services/kafka_producer_service.go   ← interface lives here (consumer package)
internal/services/impl/kafka_producer_service.go ← implementation lives here
```

### Keep interfaces small

```go
// ✅ Focused interface
type EmployeeRepository interface {
    Create(e models.Employee) (*models.Employee, error)
    FindAll() ([]models.Employee, error)
    FindById(ID string) (models.Employee, error)
    DeleteById(ID string) (int64, error)
    Update(e models.Employee) (int64, error)
}
```

### Define local interfaces for external dependencies (Kafka, HTTP clients)

This is the pattern used in `kafka_consumer_service_impl.go`:

```go
// Define a minimal interface over kafka.Consumer — never depend on the concrete type
type KafkaConsumer interface {
    SubscribeTopics(topics []string, rebalanceCb kafka.RebalanceCb) error
    ReadMessage(timeout time.Duration) (*kafka.Message, error)
    Close() error
}

type KafkaProducer interface {
    Produce(msg *kafka.Message, deliveryChan chan kafka.Event) error
    Close()
}
```

This makes the service fully testable without a real Kafka broker.

## 4. Error Handling

### Return errors — never panic in service or repository code

```go
// ✅ Return error to caller
func (er EmployeeRepositoryImpl) FindById(ID string) (models.Employee, error) {
    var e models.Employee
    err := er.db.QueryRow(sql, ID).Scan(&e.Id, &e.FirstName ...)
    if err != nil {
        return e, err   // caller decides how to handle
    }
    return e, nil
}

// ❌ Do not panic on serialisation errors in a service
value, err := json.Marshal(employee)
if err != nil {
    panic(err)  // ← replace this with: return fmt.Errorf("marshal employee: %w", err)
}
```

### Wrap errors with context using `fmt.Errorf("%w", err)`

```go
if err := kSrv.consumer.SubscribeTopics(topics, nil); err != nil {
    return fmt.Errorf("failed to subscribe to topics: %w", err)
}
```

### HTTP status mapping in controllers

| Condition | Status |
|---|---|
| Bad request body / validation failure | `400 Bad Request` |
| Resource not found | `404 Not Found` |
| Service / DB error | `500 Internal Server Error` |
| Successful creation | `201 Created` |
| Successful read/update/delete | `200 OK` |

```go
func (eCtrl EmployeeController) CreateEmployee(c echo.Context) error {
    req := dto.EmployeeRequest{}
    if err := c.Bind(&req); err != nil {
        return c.String(http.StatusBadRequest, err.Error())
    }
    if err := utils.CreateValidator().Struct(req); err != nil {
        return c.String(http.StatusBadRequest, err.Error())
    }
    result, err := eCtrl.employeeService.CreateEmployee(req)
    if err != nil {
        return c.String(http.StatusInternalServerError, err.Error())
    }
    return c.JSON(http.StatusCreated, result)
}
```

## 5. Repository Pattern (PostgreSQL)

### Use parameterised queries — never string-concatenate user input

```go
// ✅
err := er.db.QueryRow(`SELECT ... FROM employees WHERE id_employee = $1`, ID).Scan(...)

// ❌ SQL injection risk
query := "SELECT ... WHERE id_employee = '" + ID + "'"
```

### Always check `rows.Err()` after iterating

```go
rows, err := er.db.Query(`SELECT ... FROM employees`)
if err != nil {
    return nil, err
}
defer rows.Close()

var employees []models.Employee
for rows.Next() {
    var e models.Employee
    if err := rows.Scan(&e.Id, &e.FirstName ...); err != nil {
        log.Printf("scan error: %v", err)
        continue
    }
    employees = append(employees, e)
}
if err := rows.Err(); err != nil {
    return employees, err
}
return employees, nil
```

### `RETURNING` for inserts

```go
sqlStatement := `
    INSERT INTO employees (id_employee, first_name, ...)
    VALUES ($1, $2, ...)
    RETURNING id_employee`

err := er.db.QueryRow(sqlStatement, e.Id, e.FirstName, ...).Scan(&e.Id)
```

## 6. Kafka Producer (employee-service)

### Topic names come from environment variables with defaults

```go
var (
    employeeInsertTopic = utils.GetEnv("KAFKA_PRODUCER_EMPLOYEE_INSERT_TOPIC", "employee-insert.v1")
    employeeUpdateTopic = utils.GetEnv("KAFKA_PRODUCER_EMPLOYEE_UPDATE_TOPIC", "employee-update.v1")
    employeeDeleteTopic = utils.GetEnv("KAFKA_PRODUCER_EMPLOYEE_DELETE_TOPIC", "employee-deletion.v1")
)
```

### Serialise messages as JSON before producing

```go
func (kSrv KafkaProducerService) SendInsert(employee dto.EmployeeMessage) error {
    value, err := json.Marshal(employee)
    if err != nil {
        return fmt.Errorf("marshal employee message: %w", err)
    }
    return kSrv.producer.Produce(&kafka.Message{
        TopicPartition: kafka.TopicPartition{
            Topic:     &employeeInsertTopic,
            Partition: kafka.PartitionAny,
        },
        Value: value,
    }, nil)
}
```

## 7. Kafka Consumer (employee-consumer)

### Fan-out to per-topic worker pools

```go
insertCh := make(chan *kafka.Message, insertWorkerCount)
updateCh := make(chan *kafka.Message, updateWorkerCount)
deleteCh := make(chan *kafka.Message, deleteWorkerCount)

startWorkerPool(&wg, employeeInsertTopic, insertWorkerCount, insertCh, kSrv.handleInsert)
startWorkerPool(&wg, employeeUpdateTopic, updateWorkerCount, updateCh, kSrv.handleUpdate)
startWorkerPool(&wg, employeeDeleteTopic, deleteWorkerCount, deleteCh, kSrv.handleDelete)
```

### Idempotency guard with `sync.Map`

```go
func messageKey(msg *kafka.Message) string {
    return fmt.Sprintf("%s:%d:%d", *msg.TopicPartition.Topic,
        msg.TopicPartition.Partition, msg.TopicPartition.Offset)
}

if _, loaded := kSrv.processedKeys.LoadOrStore(key, struct{}{}); loaded {
    log.Printf("duplicate message %s, skipping", key)
    return
}
```

### Exponential backoff retry

```go
func (kSrv *KafkaConsumerServiceImpl) withRetry(fn func() error) error {
    backoff := kSrv.retryInitialBackoff
    for i := 0; i < kSrv.retryMaxAttempts; i++ {
        if err := fn(); err == nil {
            return nil
        } else if i == kSrv.retryMaxAttempts-1 {
            return err
        }
        time.Sleep(backoff)
        backoff = min(backoff*2, 30*time.Second)
    }
    return nil
}
```

### Dead Letter Topic (DLT) on final failure

```go
func (kSrv *KafkaConsumerServiceImpl) produceToDLT(dltTopic string, originalMsg *kafka.Message, finalErr error) {
    dltMsg := &kafka.Message{
        TopicPartition: kafka.TopicPartition{Topic: &dltTopic, Partition: kafka.PartitionAny},
        Value:          originalMsg.Value,
        Key:            originalMsg.Key,
        Headers: append(originalMsg.Headers,
            kafka.Header{Key: "x-retries",        Value: []byte(strconv.Itoa(kSrv.retryMaxAttempts))},
            kafka.Header{Key: "x-error-message",  Value: []byte(finalErr.Error())},
            kafka.Header{Key: "x-original-topic", Value: []byte(*originalMsg.TopicPartition.Topic)},
        ),
    }
    kSrv.producer.Produce(dltMsg, nil)
}
```

Topic naming convention: `<original-topic>.dlt` (e.g., `employee-insert.v1.dlt`).

## 10. Testing

### Unit test file placement

Co-locate tests with the file under test:

```
internal/services/impl/employee_service_impl.go
internal/services/impl/employee_service_impl_test.go   ← same package, or _test suffix
```

### Use stubs, not mocks

Implement the interface with a hardcoded stub:

```go
// internal/services/stubs/kafka_producer_service_stub.go
type KafkaProducerServiceStub struct {
    SendInsertCalled bool
    SendInsertErr    error
}

func (s *KafkaProducerServiceStub) SendInsert(e dto.EmployeeMessage) error {
    s.SendInsertCalled = true
    return s.SendInsertErr
}
```

### Use `go-sqlmock` for repository tests

```go
db, mock, _ := sqlmock.New()
defer db.Close()

mock.ExpectQuery(`SELECT .+ FROM employees`).
    WillReturnRows(sqlmock.NewRows([]string{"id_employee", "first_name", ...}).
        AddRow("1", "John", ...))

repo := NewEmployeeRepository(db, false)
employees, err := repo.FindAll()
assert.NoError(t, err)
assert.Len(t, employees, 1)
assert.NoError(t, mock.ExpectationsWereMet())
```

### Test the full handler with Echo's test helpers

```go
e := echo.New()
req := httptest.NewRequest(http.MethodPost, "/api/employee/", strings.NewReader(body))
req.Header.Set(echo.HeaderContentType, echo.MIMEApplicationJSON)
rec := httptest.NewRecorder()
c := e.NewContext(req, rec)

ctrl := controllers.NewEmployeeController(&stubs.EmployeeServiceStub{})

assert.NoError(t, err)
assert.Equal(t, http.StatusCreated, rec.Code)
```

## 11. Shared (`common/`) Module

### What belongs in `common/`

| Package | Contents |
|---|---|
| `common/dto` | Request/response structs shared by both services (`EmployeeRequest`, `EmployeeMessage`, `JWTResponse`) |
| `common/constants` | Cross-service enums/constants (e.g., `EmployeeStatus`) |
| `common/services/auth` | `OAuthService` interface + Keycloak implementation |
| `common/utils` | `GetEnv`, `ParseIntEnv`, `ParseBoolEnv`, `CreateValidator`, header/body helpers |

### What does NOT belong in `common/`

- Any service-specific business logic
- Any DB models (`internal/models/` per service)
- Any Echo middleware configuration

### Adding to `common/`

Always run `go mod tidy` in both consuming services after changing `common/`:

```bash
cd common && go mod tidy
cd ../employee-service && go mod tidy
cd ../employee-consumer && go mod tidy
```

## 12. Thread Safety

### Protect shared read/write maps with `sync.RWMutex`

The `KeycloakOAuthServiceImpl` pattern:

```go
type KeycloakOAuthServiceImpl struct {
    publicKeys map[string]*rsa.PublicKey
    mu         sync.RWMutex
}

// Read path
func (k *KeycloakOAuthServiceImpl) getKey(kid string) (*rsa.PublicKey, bool) {
    k.mu.RLock()
    defer k.mu.RUnlock()
    key, ok := k.publicKeys[kid]
    return key, ok
}

// Write path
func (k *KeycloakOAuthServiceImpl) storeKey(kid string, key *rsa.PublicKey) {
    k.mu.Lock()
    defer k.mu.Unlock()
    k.publicKeys[kid] = key
}
```

### Use `sync.Map` for concurrent append-only key sets

```go
// processedKeys tracks inflight/processed Kafka message keys
var processedKeys sync.Map

if _, loaded := processedKeys.LoadOrStore(key, struct{}{}); loaded {
    return // already processing
}
defer processedKeys.Delete(key) // clean up after processing
```

### goroutines must always have a WaitGroup or done channel

```go
var wg sync.WaitGroup
for i := 0; i < workerCount; i++ {
    wg.Add(1)
    go func() {
        defer wg.Done()
        for msg := range ch {
            handle(msg)
        }
    }()
}
// Signal done by closing ch, then:
wg.Wait()
```

## 13. Naming Conventions

| Element | Convention | Example |
|---|---|---|
| Files | `snake_case.go` | `employee_repository_impl.go` |
| Packages | lowercase, single word | `package repositories` |
| Interfaces | PascalCase, no `I` prefix | `EmployeeRepository` |
| Concrete types | PascalCase + `Impl` suffix | `EmployeeRepositoryImpl` |
| Test stubs | PascalCase + `Stub` suffix | `EmployeeRepositoryStub` |
| Constructors | `New<Type>(...)` | `NewEmployeeRepository(...)` |
| Methods on value receivers | short struct name + method | `(er EmployeeRepositoryImpl) Create(...)` |
| Unexported helpers | camelCase | `messageKey(msg)`, `isConsumerEnabled()` |
| Constants / env defaults | UPPER_SNAKE | `"employee-insert.v1"` |

## 14. Common Pitfalls to Avoid

| ❌ Anti-pattern | ✅ Correct approach |
|---|---|
| `panic` on serialisation error | Return `fmt.Errorf("marshal: %w", err)` |
| `os.Getenv` in business logic | `utils.GetEnv("KEY", "default")` |
| Importing `internal/` across services | Put shared types in `common/` |
| Editing `docs/*.go` / `docs/swagger.*` | Run `swag init` to regenerate |
| `log.Fatal` inside library code | Return error; only `log.Fatal` in `main` or `app.Run` |
| Storing secrets in `.env` committed to git | Use k8s Secrets or Vault in production |
| Raw string concat in SQL | Always use `$N` positional parameters |
| Unbounded goroutine spawning | Use a fixed-size worker pool with a buffered channel |
